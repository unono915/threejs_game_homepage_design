# Three.js 게임형 홈페이지 제작 설계 가이드

## 1. 프로젝트 목표

이 프로젝트는 일반적인 정적 홈페이지가 아니라, 사용자가 3D 공간을 탐색하면서 콘텐츠를 확인하는 **게임형 웹사이트**를 제작하는 것을 목표로 한다.

최종 결과물은 다음 기능을 갖춘다.

- 3D 공간이 화면에 표시된다.
- 사용자는 키보드로 캐릭터 또는 카메라를 이동할 수 있다.
- 공간 안의 오브젝트를 클릭하면 정보 패널이 열린다.
- 각 오브젝트는 홈페이지의 메뉴 역할을 한다.
- 완성된 사이트는 Vercel 또는 Netlify에 배포한다.

## 6. 사용자 경험 흐름

1. 사용자가 사이트에 접속한다.
2. 3D 공간이 로딩된다.

8. 사용자는 패널에서 자세한 설명을 읽거나 링크 버튼을 클릭한다.

## 9. 주요 기능별 구현 방향

## 9-1. 3D Canvas 구성

`@react-three/fiber`의 `Canvas`를 사용한다.

구성 요소:

- 카메라
- 조명
- 바닥
- 오브젝트
- 컨트롤

예상 구조:

```tsx
<Canvas camera={{ position: [0, 4, 7], fov: 50 }}>
  <ambientLight intensity={0.6} />
  <directionalLight position={[5, 10, 5]} intensity={1} />
  <Scene />
</Canvas>
```

---

## 9-2. 바닥과 공간 구성

처음에는 복잡한 모델 대신 기본 도형으로 구성한다.

- 바닥: plane
- 건물: box
- 메뉴 오브젝트: box 또는 sphere
- 표지판: Text 컴포넌트

이 방식으로 먼저 완성한 뒤, 추후 Blender 모델을 추가한다.

---

## 9-3. 클릭 가능한 오브젝트

각 오브젝트는 다음 기능을 가진다.

- 마우스를 올리면 커서 변경
- 클릭하면 정보 패널 열기
- 약간 떠 있거나 회전하는 애니메이션
- 오브젝트 위에 제목 표시

구현 방향:

```tsx
<mesh onClick={() => onSelect(object)}>
  <boxGeometry />
  <meshStandardMaterial color={object.color} />
</mesh>
```

---

## 9-4. 정보 패널

React 일반 UI로 만든다.  
3D Canvas 위에 겹쳐서 표시한다.

표시 내용:

- 제목
- 설명
- 버튼
- 닫기

주의할 점:

- 정보 패널은 3D 내부가 아니라 HTML UI로 구현하는 편이 쉽다.
- 모바일에서는 하단 패널 형태가 보기 좋다.

---

## 9-5. 이동 방식

초기 버전에서는 두 가지 방식 중 하나를 선택한다.

### A안: 카메라 이동형

사용자가 직접 캐릭터를 보는 것이 아니라, 카메라가 이동한다.

장점:

- 구현이 쉽다.
- 모델이 필요 없다.
- 첫 프로젝트에 적합하다.

단점:

- 실제 게임 캐릭터 느낌은 약하다.

### B안: 캐릭터 이동형

캐릭터 모델이 있고, 카메라가 캐릭터를 따라간다.

장점:

- 게임 같은 느낌이 강하다.

단점:

- 캐릭터 모델, 애니메이션, 충돌 처리가 필요하다.
- 구현 난이도가 높다.

### 권장

처음에는 **A안: 카메라 이동형**으로 구현한다.  
이후 완성되면 캐릭터 모델을 추가한다.

## 17. 주의사항

### 성능

3D 모델이 많아지면 사이트가 느려질 수 있다.

주의할 점:

- GLB 파일 용량 줄이기
- 텍스처 크기 줄이기
- 조명과 그림자 과도하게 사용하지 않기
- 모바일 성능 고려하기

### 저작권

외부 3D 모델, 이미지, 음악을 사용할 경우 라이선스를 확인해야 한다.

권장 리소스:

- 직접 제작한 모델
- 무료 라이선스가 명확한 모델
- 출처 표기가 가능한 리소스

### 보안

정적 배포 사이트에 개인정보를 넣으면 안 된다.

금지 예시:

- 학생 이름 목록 JSON
- 연락처
- 계정 정보
- API Key
- 비공개 문서
- 학교 내부 자료

필요한 데이터는 Firebase, Supabase 같은 데이터베이스에 두고 인증 후 가져오는 구조로 설계한다.
